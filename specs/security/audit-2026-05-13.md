## OWASP A01-A10 manual review

| # | Section | Status | Notes |
|---|---------|--------|-------|
| A01 | Broken access control | ✓ | `etc/acl.xml` declares `EJOsterberg_OpenSalesTax::config` under the Magento Tax config chain; `etc/adminhtml/system.xml` references it via `<resource>EJOsterberg_OpenSalesTax::config</resource>`. No `routes.xml`, no `Controller/` directory, no public-facing endpoints added. Admin form inherits Magento's default form_key CSRF protection. |
| A02 | Cryptographic failures | ✓ | API token stored encrypted via `Magento\Config\Model\Config\Backend\Encrypted` (declared on the `api_token` field in `system.xml`). Decryption goes through `Magento\Framework\Encryption\EncryptorInterface::decrypt()` in `Model\Config::getApiToken()` — that method is documented "Never log the return value." No hardcoded keys/tokens in source (`grep -rE 'eyJ\|sk_live\|pk_live\|Bearer [A-Za-z0-9]'` returns 0). TLS verification not disabled (no `CURLOPT_SSL_VERIFYPEER` override in `OstaxClient::configureCurl`). |
| A03 | Injection | ✓ | OST API requests are JSON-serialized via `Magento\Framework\Serialize\Serializer\Json::serialize()` — no string concatenation of customer input into the request URL or body. No raw SQL (no `->query(`, no `->rawQuery(`). Log statements use structured array context, not interpolated strings. No `eval`, `create_function`, `shell_exec`, or backticks. |
| A04 | Insecure design | ✓ | Fail-soft default (constitution §8): `QuoteTotalsTaxPlugin::beforeCollect` catches engine errors and falls back to Magento's tax tables. Fail-hard opt-in via `osstax/general/fail_hard=1` rethrows a `RuntimeException` Magento surfaces during checkout. Non-USD currency and non-US country short-circuit before any engine call (constitution §5). Both paths exercised by unit tests `QuoteTotalsTaxPluginTest::testBeforeCollectFailSoftSwallowsEngineError`, `::testBeforeCollectFailHardRethrows`, `::testBeforeCollectSkipsWhenCurrencyIsNotUsd`, `::testBeforeCollectSkipsWhenCountryIsNotUs`. |
| A05 | Security misconfiguration | ✓ (with deferred URL backend revalidation) | URL field has `<validate>validate-url</validate>` frontend class. **Deferred to v0.2:** backend re-validation via a custom backend_model with private-IP-range allowlist (currently any URL the admin enters is accepted; merchants self-hosting on localhost/private network is the supported pattern, so the gap is intentional but should be documented). No `display_errors` injected; no CORS opened. Exception messages from `OstaxClient` carry HTTP status only, never the engine URL or token. |
| A06 | Vulnerable & outdated components | ✓ | `composer audit` clean. PHP constraint `^8.1` covers all currently-supported PHP minors. Module-level `composer.json` constraints (`magento/framework: ^103.0`) match the constitution's 2.4.6+ target. |
| A07 | Identification & authentication failures | N/A | Module adds no auth surface — customer requests run inside Magento's existing session/CSRF lifecycle, admin config inherits Magento's admin session. |
| A08 | Software & data integrity failures | ✓ | `composer.lock` intentionally not committed (we are a library, not an application). No `post-install-cmd` / `post-update-cmd` scripts in `composer.json`. CI installs dev dependencies for tests; no Marketplace signing yet (deferred to v1.1). |
| A09 | Security logging & monitoring failures | ✓ | All tax-calc paths log structured metadata (`quote_id`, `line_count`, `http_status`, `rtt_ms`) via `Psr\Log\LoggerInterface` — Magento's injected logger. **Customer addresses and full quote payloads never appear in log statements.** API token decrypted in memory only; never logged. Engine errors logged at WARN/ERROR with HTTP status. |
| A10 | Server-side request forgery | Partial (intentional for alpha) | `api_url` is validated at save time via the `validate-url` frontend class. **Deferred to v0.2:** private-IP-range allowlist (current design intentionally permits localhost / private IPs so merchants can self-host the engine alongside Magento on the same VM). Outbound Curl timeout set to 10s via `setTimeout(self::DEFAULT_TIMEOUT_SECONDS)` so a malicious / slow URL cannot stall a checkout indefinitely. No request-time interpolation of merchant-controlled input beyond the configured base URL + the fixed `/v1/*` paths. |
